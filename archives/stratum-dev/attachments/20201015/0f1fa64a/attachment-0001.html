<p class="MsoNormal" style="line-height:13.5pt">Hi Narada,<br>
<br>
The specific issue that you are seeing has to do with the way that protobufs are being initialized. When a binary or shared library is loaded that contains a protobuf, initialization code like the code below is executed.
<o:p></o:p></p>
<div>
<p class="MsoNormal" style="line-height:13.5pt"><o:p> </o:p></p>
</div>
<div>
<p class="MsoNormal" style="line-height:13.5pt"><span style="font-family:"Courier New"">// Force running AddDescriptors() at dynamic initialization time.</span><o:p></o:p></p>
</div>
<div>
<p class="MsoNormal" style="line-height:13.5pt"><span style="font-size:12.0pt;font-family:"Courier New";color:#222222;background:white">static bool dynamic_init_dummy_google_2frpc_2fcode_2eproto =<o:p></o:p></span></p>
</div>
<p class="MsoNormal" style="line-height:13.5pt"><o:p> </o:p></p>
<div>
<p class="MsoNormal" style="line-height:13.5pt"><span style="font-size:12.0pt;font-family:"Courier New";color:#222222;background:white">(static_cast<void>(::PROTOBUF_NAMESPACE_ID::internal::AddDescriptors(&descriptor_table_google_2frpc_2fcode_2eproto)), true);</span><o:p></o:p></p>
</div>
<p class="MsoNormal" style="line-height:13.5pt"><br>
This likely means that libprotobuf, libgrpc, and any other library that introduces a protobuf (e.g. google/rpc/[code,status], P4Runtime) needs to be either linked in the stratum_bfrt binary, linked to bf-drivers, OR loaded once as a shared library for bf-drivers
 and stratum_bfrt to share.<o:p></o:p></p>
</div>
<div>
<p class="MsoNormal" style="line-height:13.5pt"><o:p> </o:p></p>
